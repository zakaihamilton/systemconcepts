## 2024-05-18 - Prevent VariableSizeList Unnecessary Re-renders
**Learning:** Using an inline object for `itemData` prop in `react-window`'s `VariableSizeList` breaks `React.memo` for the list items (e.g. `SearchResultItem`), causing them to re-render on every list update, scroll, or state change, regardless of whether their props have actually changed.
**Action:** Always wrap `itemData` and its dependencies (like functions e.g. `gotoArticle`) using `useMemo` and `useCallback` when using `react-window` to ensure that item components are not unnecessarily re-rendered.
## 2024-05-18 - Prevent FixedSizeList/VariableSizeList Unnecessary Item Re-renders During Scrolling
**Learning:** Using `React.memo` alone on list item components in `react-window` or custom virtualized lists (like `FixedSizeList`) does not prevent re-renders during scrolling. This is because the list component generates a new `style` object reference on every scroll event (even if the left/top properties are identical for a stationary visible item), which breaks the shallow comparison of `React.memo`.
**Action:** Always provide a custom `arePropsEqual` comparator function to `React.memo` for virtualized list item components. This comparator should deeply compare the `style` object keys and values instead of relying on object reference equality, ensuring that items only re-render when their actual positional data or `itemData` changes.
